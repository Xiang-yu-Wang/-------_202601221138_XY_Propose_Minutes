### 管理後臺

#### 訪問方式
- **開發環境**: `http://localhost:5173/#/admin`
- **正式網站**: `網站域名/#/admin`
- **密碼**: `admin123`

#### 主要功能

1. **公告管理**
   - 點擊「新增公告」
   - 填寫標題、內容、類型
   - 點擊「新增」立即生效

2. **產品管理**
   - 切換到「產品管理」頁籤

   - 上傳圖片（支援拖放）
   - 填寫價格與描述

3. **GitHub 同步** 🔥
   - 設定 GitHub Token（一次性）
   - 編輯完內容後點擊「同步到 GitHub」
   - 等待 1-2 分鐘自動部署完成
   - 刷新網站即可看到更新

## 🔄 GitHub 同步流程（重點功能）

### 設定步驟（一次性）

1. **取得 GitHub Token**
   ```
   GitHub → Settings → Developer settings
   → Personal access tokens → Tokens (classic)
   → Generate new token
   → 勾選 repo 權限
   → 複製 Token
   ```

2. **設定 Token 到管理後臺**
   ```
   管理後臺 → 點擊「GitHub Token 設定」
   → 貼上 Token → 儲存
   ```

### 日常使用

```
編輯公告/產品 → 點擊「同步到 GitHub」 → 確認 → 等待 1-2 分鐘 → 完成
```

## 🎓 使用建議

### 給管理員

1. **定期備份**
   - 每週導出一次公告和產品備份
   - 保存 JSON 檔案到本地

2. **圖片優化**
   - 使用線上壓縮工具（TinyPNG）
   - 圖片寬度控制在 300-600px
   - 保持檔案大小在 500KB 以下

3. **內容更新頻率**
   - 公告：有重要訊息時即時更新
   - 產品：週期性檢查上下架狀態

4. **GitHub 同步**
   - 批量編輯完成後再同步
   - 避免頻繁同步（建議間隔 30 分鐘以上）

### 給開發者

1. **代碼規範**
   - 使用 TypeScript 嚴格模式
   - 遵循 Vue 3 Composition API 最佳實踐
   - 使用 Tailwind CSS 而非自訂樣式

2. **新功能開發**
   - 在 feature 分支開發
   - 測試後再合併到 main
   - 保持 commit 訊息清晰

3. **環境管理**
   - 開發環境使用 `.env`
   - 生產環境使用 `.env.production`
   - 不要提交敏感資訊
