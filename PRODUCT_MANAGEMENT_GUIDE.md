# 產品管理系統使用指南

## 🎉 功能概覽

已成功在管理後台添加**產品管理**功能,老板現在可以自行上傳和管理販售的物品!

## 📍 訪問方式

1. 開啟管理後台: `/admin`
2. 輸入密碼登入(預設: `admin123`)
3. 點擊頂部的**「產品管理」**頁籤

## ✨ 主要功能

### 1. 新增產品

點擊「新增產品」按鈕,填寫表單:

- **產品名稱** - 例如:高級禮盒組合
- **分類** - 例如:禮品盒、食品、飲品
- **價格** - 輸入數字(單位:元)
- **產品描述** - 詳細說明產品特色
- **產品圖片** - 上傳圖片(限制 500KB)
  - 支援格式: JPG, PNG, GIF, WebP
  - 圖片會自動轉為 Base64 儲存
  - 建議尺寸: 300x200 px
- **立即上架** - 勾選後立即在網站顯示

### 2. 編輯產品

- 點擊產品卡片的「編輯」按鈕
- 修改任何欄位
- 重新上傳圖片(選填)
- 點擊「更新」儲存

### 3. 上架/下架

- 點擊「上架」或「下架」按鈕
- 下架的產品不會在前台顯示
- 但資料仍保留,可隨時重新上架

### 4. 刪除產品

- 點擊紅色的垃圾桶圖示
- **注意**: 刪除後無法復原,建議先導出備份

### 5. 匯出備份

- 點擊「導出」按鈕
- 下載 JSON 檔案(`products_YYYY-MM-DD.json`)
- 建議定期備份,避免資料遺失

### 6. 匯入資料

- 點擊「導入」按鈕
- 選擇之前導出的 JSON 檔案
- 系統會驗證格式後覆蓋現有資料

### 7. 重置為預設

- 點擊「重置」按鈕
- 恢復到系統預設的產品列表
- **警告**: 會清除所有自訂產品,請先備份!

## 📊 統計資訊

管理頁面頂部會顯示:

- **總產品數** - 所有產品總數
- **上架中** - 目前在前台顯示的產品數
- **已下架** - 暫停販售的產品數
- **分類數** - 有幾種產品分類
- **總價值** - 所有產品價格總和

## 🖼️ 圖片處理建議

### 為什麼限制 500KB?

- localStorage 總容量限制約 5-10MB
- 圖片轉 Base64 會增大約 33%
- 500KB 圖片轉換後約 670KB
- 可儲存約 10-15 張產品圖

### 如何壓縮圖片?

**線上工具:**
- [TinyPNG](https://tinypng.com) - 推薦!自動壓縮
- [Squoosh](https://squoosh.app) - Google 開發
- [CompressJPEG](https://compressjpeg.com)

**建議設定:**
- 尺寸: 400x300 px 或 600x400 px
- 格式: JPEG (品質 80%)
- 檔案大小: < 300KB 最佳

**壓縮步驟:**
1. 上傳原始圖片到壓縮工具
2. 調整品質到 80%
3. 下載壓縮後的圖片
4. 在管理後台上傳

## 💾 資料儲存說明

### 儲存位置
- 使用瀏覽器 **localStorage**
- 資料存在本機,不會上傳到伺服器
- 只在當前瀏覽器/設備有效

### 注意事項
⚠️ **重要限制:**
- 清除瀏覽器資料會遺失產品
- 換電腦/瀏覽器無法同步
- 建議定期使用「導出」功能備份

✅ **最佳實踐:**
1. **每週導出備份一次**
2. **更新產品後立即備份**
3. **將 JSON 檔存到雲端硬碟**
4. **多台電腦使用導入功能同步**

## 🔄 跨設備同步方案

### 手動同步(推薦)
1. 在電腦 A 編輯產品後點擊「導出」
2. 將 JSON 檔案存到 Google Drive / Dropbox
3. 在電腦 B 下載 JSON 檔案
4. 在電腦 B 的管理後台點擊「導入」

### 定期備份流程
```
每週一:
1. 開啟管理後台
2. 導出產品資料
3. 上傳到雲端硬碟 /備份/products/ 資料夾
4. 檔名範例: products_2026-01-27.json
```

## 🐛 常見問題

### Q: 圖片上傳失敗「超過 500KB」怎麼辦?
A: 使用 TinyPNG 等工具壓縮圖片後再上傳

### Q: 我的產品突然消失了?
A: 可能清除了瀏覽器資料。檢查是否有備份的 JSON 檔案可以匯入

### Q: 可以上傳影片嗎?
A: 不行,只支援圖片。影片太大會超出 localStorage 限制

### Q: 如何批量上傳 50 個產品?
A: 
1. 先手動建立 1-2 個產品範例
2. 導出 JSON 檔案
3. 複製 JSON 格式,用 Excel 批量產生
4. 使用「導入」功能上傳

### Q: 前台沒有顯示新產品?
A: 檢查以下項目:
- ✅ 產品是否勾選「立即上架」
- ✅ 重新整理前台頁面(Ctrl + F5)
- ✅ 檢查瀏覽器 Console 是否有錯誤

## 🔐 安全建議

1. **修改管理後台密碼**:
   - 編輯 `.env` 檔案
   - 設定 `VITE_ADMIN_PASSWORD=你的新密碼`
   - 重新執行 `bun run dev`

2. **不要分享備份檔案**:
   - JSON 檔案包含所有產品資料
   - Base64 圖片可被還原

3. **定期檢查產品列表**:
   - 確認沒有被誤刪
   - 價格是否正確
